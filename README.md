# Force Paste — Raycast Extension

Type clipboard content character by character to bypass paste restrictions.

Some fields (bank portals, remote desktop clients, legacy forms) block `⌘V`. This extension works around that by simulating individual keystrokes via macOS Accessibility APIs — as if you typed every character by hand.

---

## Commands

### Smart Force Paste

Reads the clipboard and types it out at a configurable speed.

- Run it once to start typing.
- **Press `Esc` to stop mid-way** — detected instantly without needing to re-open Raycast.
- Run it again while typing is in progress to stop (toggle behavior).

#### Preferences

| Preference | Default | Description |
|---|---|---|
| Words Per Minute | `60` | Typing speed. 1 word = 5 characters. Sets keystroke delay when min/max are `0`. |
| Min Delay (ms) | `0` | Minimum delay between keystrokes. `0` = derived from WPM. |
| Max Delay (ms) | `0` | Maximum delay between keystrokes. `0` = derived from WPM. |

### Stop Force Paste

Stops an in-progress typing session from anywhere. Useful if you assigned a hotkey to this command.

---

## Requirements

- macOS
- **Accessibility permission** — required for simulated keystrokes.  
  Grant it at: System Settings → Privacy & Security → Accessibility → Raycast

---

## How the Esc-to-stop works

A lightweight Python3 background process polls `CGEventSourceKeyState` (hardware-only key state, no Input Monitoring permission needed) for the Escape key while typing is active. When detected, it writes a sentinel file that the typing loop checks between each character and stops cleanly.

Synthetic keystrokes generated by the extension itself never appear in the hardware state, so they cannot accidentally trigger a stop.
